Método POST · RFC 2616 §9.5

Envío de datos al servidor

Este formulario envía una petición POST al recurso /submit. Permite verificar que el TWS lee correctamente el Content-Length y el cuerpo de la petición, y que el PIBL reenvía el body completo al backend elegido por Round Robin.

Formulario

Cancelar

Cómo probarlo desde curl

curl -v -X POST http://<IP_PIBL>:8080/submit \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "nombre=Felipe&mensaje=Hola"

Probar errores

  • GET /no-existe.html → debe responder 404 Not Found
  • Petición malformada por telnet → debe responder 400 Bad Request
El TWS debe procesar el body completo según Content-Length y responder con 200 OK.
