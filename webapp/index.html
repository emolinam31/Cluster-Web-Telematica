Servidor web de prueba para PIBL + TWS

Esta aplicación se replica en tres servidores backend (TWS) y se accede a través del Proxy Inverso + Balanceador de Carga (PIBL) implementado en C usando la API Sockets. Cada página cubre uno de los casos de prueba exigidos por el proyecto.

¿Qué se prueba aquí?

  • Servir text/html, text/css, image/jpeg, image/png y application/octet-stream.
  • Manejo correcto de rutas con subdirectorios (/css/, /img/, /files/).
  • Concurrencia: el navegador pide múltiples recursos en paralelo.
  • Métodos GET, HEAD y POST (ver formulario).
  • Errores: 404 (recurso inexistente) y 400 (petición malformada).
Servido por ./server <PORT> <LogFile> <DocRoot> detrás del PIBL en AWS EC2.
